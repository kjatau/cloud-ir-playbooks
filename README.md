# Cloud Incident Response Playbooks

Step-by-step incident response playbooks for AWS security alerts.
Built for security teams responding to GuardDuty, CloudTrail, and IAM alerts.

## Playbooks
- 01 — Brute Force Attack Detected
- 02 — Exposed S3 Bucket Detected
- 03 — Root Account Login Detected
- 04 — Suspicious API Activity Detected
- 05 — Crypto Mining Activity Detected

## How to use
Each playbook covers:
- Trigger condition
- Severity and response time target
- Triage steps
- Containment commands
- Investigation steps
- Recovery actions
- Documentation requirements

## Tools referenced
- AWS GuardDuty
- AWS CloudTrail
- AWS IAM
- AWS EC2
- AWS S3
